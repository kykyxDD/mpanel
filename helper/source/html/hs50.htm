Measurements

 

On this page the user enters the site measurements, and defines edge and corner details.

 

 

  • Number of connections.

Sets this page up for 3, 4, 5 or 6 connection points. 

 

  • Measurements

 

Between connection points 

 

It is always best to make site measurements between the actual connection points (i.e.- at the pole tops) as this minimises errors due to pole lean and connection details. 

 

The measurements are taken between the inside edges of the fixing points, perhaps with a tape measure with a hook on the end. It is essential, that the tape measure form a straight line between the connection points and that it is pulled tight. 

 

(Do not bend the tape around obstacles such as walls or trees as this will produce inaccurate measurements. If obstacles prevent straight-line measurements to be taken you will need to take additional measurements sufficient to allow calculation of the correct dimensions. Refer to separate note "how to measure around obstacles")

 

 

Between pole bottoms

 

A less accurate option is to enter measurements taken at ground level. In this case the fixing point should be allowed for at ground level, perhaps by drawing it with chalk on the ground. A check should also be made to see if the posts are vertical.

 

 

 

Regular shapes

 

For initial designs and estimations you can create regular shaped sails. These will produce realistic visualisations and material estimations. But since the measurements are not the real on site dimensions between the poles, they should not be used to make production panels. 

 

 

 

The measurements can be cleared, which will be the usual start of a measuring session. 

 

Once a set of measurements has ben entered you may want to save them into a named file for future use. You can do this with the "File   Save as" menu item

 

Example measurements

For ease of practising with the program a set of example measurements can be loaded in that will work to produce a reasonable sail design, for 3, 4, 5 and 6 poles.

 

 

 

 

  • Edges

Hem type          set the hem width for this edge

Dip/Span ratio    this will set the tightness of the cable or webbing edge for this edge. 

Fixed edge        specifies if the edge is restrained to a fixed edge, like a kedar edge against a building

Mid point         adds a support at the midpoint

 

Note:   if the sail has a continuous cable running though the corner connections, like an pulley or ezglide, tick the "Force same tension on all edges" . 

This will alter the dip/span ratios on each side, to maintain the same tension in each cable edge, with the average dip/span being as specified. 

Typically this means that longer edges will have a larger dip/span, and shorter edges a small dip/span, than the specified dip/span value.

 

 

 

  • Corners

Corner hardware   specify the corner hardware to be used

Link type         specify the type of link to be used

Link length       this value is normally read from the link database. 

                  But for guys you can specify the length in the table. 

 

 

 

  • Details

Warp direction    specifies the warp direction, ie what direction the model should be panelled in.

Lofting %         specifies the amount of lofting, ie how much the sail design raised above the natural position.

Tag corner        specifies the corner that will be marked, usually with the manufacturers label

Number of panels  specifies the number of panels created, presently 1, 2, 3, or 4.

Straighten seams  specifies that the program will try to make the seams as straight as possible.

 

The warp direction is defined when the actual model matches the picture in orientation. 

So for a 4 pole sail, vertical warp is from the midpoint of AB to the midpoint of CD

Diagonal warp direction can also be specified. This can be used to obtain a seam that runs between the high poles, for example. 

 

Lofting can be applied to counter the effects of the sail sagging over time. Lofting is achieved by shaping the panels so that more tension is applied between the high points. The lofting tension is scaled so that a setting of 20% will raise the centre of a square hypar 20% towards the high diagonal.

 

Lofting does not apply to 3 sided sails.

On 4 sided sails lofting is applied toward the high diagonal, whichever it is.

On 5 sided sails we assume that corner B is high, and apply lofting towards BD and BE diagonals.

On 6 sided sails we assume that either A, C, E are high, or B, D, F are high, and apply lofting accordingly.

 

 

Notes on seam straightening.

 

MPSD by default produces seams that are evenly spaced across the model. This gives a natural look to the seams, keeps the panels locally the same width as thier neigbours, and avoids mid parts of the panels from becomming very narrow. 

 

To lower production costs there is a requiremet for straighter seams. These seams may not look as nice, and may be placed close to the model edge. It is up to the user to judge if the seam staightning that MPSD applies is satisfactory for the particular job. 

 

 

Notes on diagonal warp.

 

Diagonal warp can be applied to 4, 5 and 6 pole structures, although it's main use is with 4 pole structures. Using diagonal seams makes a seam that splits the corner angle on two of the corners. This effectively halves the panel corner angle, which can lead to problems when the hem allowance extends the panel corners excessively.

 

The default diagonal warp seam is a subtle curve that ensures that the corner angle is divided evenly between the 2 panels. This curve is no problem if the panel is cut or plotted automatically. If the curve is created with a flexible straight edge with a centre dip value, it will only be an approximation to the subtle curve. In this case the user must judge if the curve is acceptable, particularly in the corners. 

 

When seam straightening  is used with diagonal warp, the corner angle is no longer divided equally between two panels, and one panel may have a very narrow corner.  The user must judge if this is acceptable for the particular job.

 

Diagonal warp is not supported with Mid poles. It could be if enough users request this feature.
