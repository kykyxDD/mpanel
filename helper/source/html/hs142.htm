Imports

 

  • Partial model file

When you use the MPSD Web App at      www.mpcalc.com      the model file emailed to you contains the site dimensions, but no details like corner hardware dimensions, hem widths, etc. This is a partial model file.

 

The partial model file can be opened in MPSD as a normal file, and will find the missing data from:

 

a)    A template file, if you have previously created one, and if the template is in the same dimensions as the partial model file 

 

b)    The MPSD default settings file.

 

 

  • Import pole top dxf

Reads a dxf file that contains a 3dpoly drawn between the pole tops, in the order A..B..C..D etc.

This information would usually come from automated measuring equipment such as the Prodim Proliner.

 

There can be exactly one 3dpoly in the dxf file, which will be used to define the pole tops.

 

Or there can be any number of 3dpolys, with exactly one 3dpoly on a layer beginning with "P"  (pole tops)

Optionally there can also be exactly one 3dpoly on a layer beginning with "G" (ground) to define the average pole bottoms. 

 

The 3dpoly will contain one point for each pole top, so valid number of points are 3, 4, 5 and 6

There is no information in a dxf about the measurement units, so you have to choose units in MPSP to match the units used for measuring.

Formatting of the units is performed, so measurements in inches can be used as feet_inches or feet_inches_fractions.

 

The dxf information is in XYZ format, so the program will calculate the equivalent pole top to pole top dimensions, and enter them in the measurement tables. This means that the measurement error, reported during calculation, will always be zero, so no real checks on the measurement accuracy are being made.
